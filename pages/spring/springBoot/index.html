    # spring boot

    • springboot的自动装配原理?
      • springApplication注解包含三个注解、分别是ComponentScan、Springbootconfiguration、EnableAutoConfiguration,其中ComponentScan注解作用是定义扫描路径，默认是springApplication注解类的当前目录，还可以通过ComponentScan注解指定其他扫描目录，配合Component注解可以定义需要springboot自动装配管理的类；EnableAutoConfiguration注解引入了AutoConfigurationImportSelector类，该类中的loadFactoryNames方法通过加载扫描springboot源码中的配置文件加载相应的配置类，并结合Condition注解条件实现智能加载，最终实现自动装配Bean。
    • springboot事务原理
      • TransactionInterceptor:负责拦截方法执行，进行判断是否需要提交或者回滚事务。
      • PlatformTransactionManager:是Spring 中的事务管理接口，真正定义了事务如何回滚和提交。
      • 原理：Spring中的@Transactional注解的事务实现主要通过TransactionInterceptor拦截器来进行实现的，拦截目标方法，然后判断异常是不是目标异常，如果是目标异常就行进行回滚，否则就进行事务提交。
    • springboot事务
      • 标记:@Transactional
      • 特点:可以标注在类或者接口上，但一般不建议标注在接口上，且只能使用在public方法上，其他开放方法会失效
      • 捕获异常:默认只能在unChecked异常下回滚，checked异常下不会回滚，如果需要回滚，需要设置异常类型@Transactional(rollbackFor=Exception.class)
      • 只读事务:启动该事务会增加线程开销，如果是读取数据可以不必开启@Transactional(propagation=Propagation.NOT_SUPPORTED,readOnly=true)
      • 事务传播模式
      • REQUIRED(必须):该模式必须运行在事务中，如果处于事务之中，则加入，否则新建事务
      • NOT_SUPPORTED(不支持):该模式不支持事务，如果处于事务之中，则会挂起当前事务，执行完成后再恢复事务
      • MANDATORY(强制):该模式必须运行在事务中，如果不在则报错
      • REQUIRESNEW（必须新的）：该模式下不管是否存在事务，都会给自己新建一个事务，存在原有事务会被挂起
      • SUPPORTS (支持):该模式支持事务，如果处于事务之中，则加入事务，如果不处于事务之中，就在该环境下运行
      • NEVER(从不):该模式不能运行在事务环境中，如果处于，则会抛错
      • NESTED(嵌套):该模式下如果处于事务环境中，则会新建一个嵌套事务，嵌套事务不会影响外面事务的回滚，如果没有事务，则新建一个事务
      • 事务不回滚原因
        • 方法非public
        • 异常不是unChecked,且没有设置异常类型
        • 数据库不支持事务
        • 是否开启对事务的解析
        • 事务传播特性是否设置正确
        • springboot是否扫描到该包文件
        • 同一个类中方法调用,如果先调用deleteUser()方法,deleteUserA方法就不会回滚,修改方式，把当前类自己注入就可以
    	 public void deleteUser() throws MyException{
    deleteUser2();
}
@Transactional
public void deleteUser2() throws MyException{
    userMapper.deleteUserA();
    int i = 1 / 0;
    userMapper.deleteUserB();
}

   @Service
public class UserService {
    @Autowired
    private UserMapper userMapper;
    //自己注入自己
    @Autowired
    UserService userService;

    public void deleteUser() throws MyException{
        userService.deleteUser2();
    }

    @Transactional
    public void deleteUser2() throws MyException{
        userMapper.deleteUserA();
        int i = 1 / 0;
        userMapper.deleteUserB();
    }
}

     - 沒有catch住异常,或者异常没有抛出

       @Transactional
public void deleteUser() {
    userMapper.deleteUserA();
    try {
        int i = 1 / 0;
        userMapper.deleteUserB();
    } catch (Exception e) {
        e.printStackTrace();
    }
}

     - 新开启线程,spring实现事务原理是通过ThreadLocal把数据库连接绑定到当前线程中,新开启一个线程获取的连接就不是同一个了

       @Transactional
public void deleteUser() throws MyException{
    userMapper.deleteUserA();
    try {
        //??1?,??deleteUserA???
        Thread.sleep(1000);
    } catch (InterruptedException e) {
        e.printStackTrace();
    }
    new Thread(() -> {
        int i = 1/0;
        userMapper.deleteUserB();
    }).start();    
}
